## Version History

### Version Numbering

RAGE follows Semantic Versioning (SemVer):

```
MAJOR.MINOR.PATCH

Example: 1.2.3
```

- **MAJOR**: Incompatible API changes
- **MINOR**: Backwards-compatible functionality additions
- **PATCH**: Backwards-compatible bug fixes

### Release Types

- **Alpha**: Early development, frequent changes
- **Beta**: Feature-complete, testing and refinement
- **RC (Release Candidate)**: Final testing before release
- **Stable**: Production-ready release

## Change Categories

Changes are categorized as:

- **Added**: New features
- **Changed**: Changes in existing functionality
- **Deprecated**: Soon-to-be removed features
- **Removed**: Removed features
- **Fixed**: Bug fixes
- **Security**: Security vulnerabilities fixed
- **Documentation**: Documentation updates
- **Performance**: Performance improvements
- **Refactoring**: Code refactoring without functional changes
